虚拟内存概述

理解虚拟内存的动机、定义与核心特征

为什么需要虚拟内存？
  • 程序规模增长：程序越来越大，物理内存无法容纳全部程序
  • 多道程序并发：多个程序同时运行，内存不够分
  • 局部性原理：程序在一段时间内只访问一小部分内存 → 不需要全部装入
    • 虚拟内存的定义

    基于局部性原理，在程序执行时只将当前需要的部分装入内存，其余部分留在外存。当访问的信息不在内存时，由OS将所需信息从外存调入内存；内存空间不够时，将内存中暂时不用的内容换出到外存。

    虚拟内存的四大特征
    ① 多次性
    作业无需一次性全部装入
    ② 对换性
    运行中可换入/换出
    ③ 虚拟性
    逻辑上扩充内存容量
    ④ 离散性
    不必连续分配内存
    ⚠️ 易错：虚拟内存的"多次性"是最核心的特征，是与传统连续分配的根本区别。"虚拟性"是结果/表现，不是本质特征（考试陷阱！）
    虚拟内存的三种实现方式
    实现方式离散分配基础换入/换出单位优缺点
    请求分页基本分页页最常用，产生内部碎片
    请求分段基本分段段便于共享/保护，产生外部碎片
    请求段页式基本段页式页结合两者优点，实现复杂
    ⚠️ 前提条件：虚拟内存的实现需要一定容量的内存和外存 + 页表机制/段表机制 + 中断机构 + 地址变换机构

    页表项结构、缺页中断机制与地址变换过程

    请求分页 vs 基本分页
  • 基本分页：程序全部装入内存后才运行，不会缺页
  • 请求分页：程序部分装入即可运行，访问到不在内存的页时产生缺页中断
  • 区别的核心：多了缺页中断和页面置换功能

    页面置换算法

    OPT / FIFO / LRU / CLOCK 算法原理与交互式模拟

    算法速览对比

    🥇 OPT 最佳置换 理论最优

    淘汰未来最长时间不再访问的页面。无法实现，仅作衡量标准。

    🥈 FIFO 先进先出 有Belady异常

    淘汰最早进入内存的页面。性能差，可能产生Belady异常。

    🥉 LRU 最近最久未用 性能好

    淘汰最近最长时间未被访问的页面。性能接近OPT，但实现开销大。

    🕐 CLOCK 时钟算法 实用折中

    简单CLOCK只看访问位A；改进型同时考虑访问位A+修改位M，按(0,0)→(0,1)→(1,0)→(1,1)优先淘汰，减少磁盘I/O。

    Belady 异常

    定义：FIFO算法中，分配给进程的物理块数增加，缺页次数反而增加的异常现象。

    原因：FIFO的替换决策与进程使用页面的历史无关，只看进入顺序。FIFO和未基于"栈算法"的算法才会出现。

    结论：LRU、OPT 基于"栈算法"，不会出现Belady异常。

    简单 CLOCK 算法

    原理

  • 为每页设置一个访问位A，将页面组织成循环链表（环形）
  • 指针循环扫描：若当前页A=0，则淘汰；若A=1，则将A置0，跳过继续扫描
  • 最多扫描一整圈后，所有A都被置0，一定能找到淘汰页
    • 核心思路：最近访问过的页(A=1)给它"第二次机会"，跳过并将A置0；未被访问(A=0)的直接淘汰

    改进型 CLOCK 算法详解

    （访问位A, 修改位M）四种页面类型

    优先级(A, M)含义淘汰选择
    第1选择(0, 0)最近未访问且未修改✅ 最佳淘汰对象
    第2选择(0, 1)最近未访问但已修改淘汰时需写回磁盘
    第3选择(1, 0)最近访问过但未修改可能还会再用
    第4选择(1, 1)最近访问过且已修改❌ 最不应淘汰

    扫描时：第1轮找(0,0)，途中将A置0 → 第2轮找(0,1)，途中将M置0 → 第3轮找(0,0) → 第4轮找(0,1)。最多4轮必能找到。

    分配与置换策略

    驻留集、分配策略、页面调入时机与来源

    驻留集

    驻留集：分配给进程的物理块集合。驻留集大小决定了内存中能保留的页面数，直接影响缺页率。

  • 驻留集太小：缺页频繁，进程频繁换页（抖动）
  • 驻留集太大：内存利用率低，并发度下降
  • 驻留集适中：缺页率可接受，CPU利用率最高
    • 三种分配策略
    策略物理块数置换范围优点缺点
    固定分配局部置换进程创建时固定仅在本进程的块中各进程互不影响块数不合理时难以调整
    可变分配全局置换可动态增减可从其他进程夺取灵活、实现简单可能干扰其他进程
    可变分配局部置换可动态增减仅在本进程的块中兼顾灵活与隔离实现复杂
    ⚠️ 高频考点："可变分配全局置换"是最常用的策略。缺页时从空闲块中分配，无空闲块时才从其他进程夺取。
    页面调入时机

    📅 预调页策略

    一次调入若干相邻页面，利用空间局部性。适用于进程首次调入时，预测困难。

    📨 请求调页策略

    访问到不在内存的页时才调入，一次只调一页。实现简单，是主要策略，但I/O开销大。

    从何处调入页面
  • 对换区足够：进程运行前从文件区复制到对换区，之后全部从对换区调入（对换区采用连续分配，I/O速度更快）
  • 对换区不够：不会修改的页面从文件区调入；可能修改的页面从对换区调入
  • UNIX方式：首次使用从文件区调入，换出时写入对换区，再次调入从对换区调入
    • ⚠️ 关键区别：文件区存放程序原文件（离散分配，I/O慢），对换区存放换出的页面（连续分配，I/O快）。优先从对换区调入。

    抖动与工作集

    理解抖动现象、工作集模型与预防策略

    抖动（Thrashing）

    定义：系统中同时运行的进程太多，每个进程分到的物理块太少，频繁发生缺页，导致CPU利用率急剧下降的现象。

    表现：进程大部分时间用于页面换入换出，CPU利用率极低，OS以为需要增加并发度 → 更多进程 → 更严重 → 恶性循环

    工作集（Working Set）

    定义：在某段时间间隔（工作集窗口 Δ）内，进程实际访问的页面集合。

  • 工作集大小随 Δ 增大而增大
  • 工作集是近期访问页面的集合，反映局部性
  • 驻留集 ≥ 工作集 → 不会抖动；驻留集 < 工作集 → 可能抖动

    预防抖动的方法
  • 采用工作集策略：保证驻留集 ≥ 工作集
  • "L = S"准则：产生缺页的平均时间(L) = 处理缺页的平均时间(S)时，CPU利用率最高
  • 选择合适的驻留集：利用缺页率反馈，动态调整物理块分配
  • 挂起部分进程：CPU利用率过低时，减少多道程序度

    缺页率影响因素与有效访问时间计算

    影响缺页率的因素

    📐 页面大小

    页面增大 → 单页含更多信息 → 缺页率先降后升（太小则缺页多，太大则浪费空间）

    🧱 物理块数

    物理块增多 → 驻留集增大 → 缺页率下降（FIFO除外，可能Belady异常）

    🔄 置换算法

    不同算法对相同引用串产生不同缺页率。OPT最优，LRU次之，FIFO最差

    📝 程序局部性

    局部性越好 → 工作集越小 → 缺页率越低。循环、顺序访问局部性好

    有效访问时间（EAT）计算

    基本公式

    EAT = (1 - p) × ma + p × 缺页处理时间

  • p = 缺页率（0 ≤ p ≤ 1）
  • ma = 访问内存时间
  • 缺页处理时间 = 缺页中断服务时间 + 换入页面时间 + （若需置换）换出页面时间 + 重新执行指令时间

    • 完整公式（含快表TLB）

    EAT = 命中TLB × (查TLB + 访内存)

         + 未命中TLB × (1-p) × (查TLB + 查页表 + 访内存)

     &;    + 未命中TLB × p × (查TLB + 查页表 + 缺页处理 + 访内存)

    ⚠️ 考试注意：计算EAT时务必统一时间单位！1ms = 1,000μs = 1,000,000ns。通常内存访问用ns，缺页处理用ms，需要换算。
    EAT 计算器
    * 若不需要换出（有空闲块），换出时间填0
    经典例题

    例题：请求分页系统EAT计算

    某请求分页系统，访问内存时间为200ns，缺页中断服务时间为10μs，从磁盘换入一个页面需8ms，换出一个修改页需15μs，缺页率为5%，其中50%的页面被修改过。求EAT。

    点击查看解答

    缺页处理时间 = 缺页中断服务 + 换入 + 换出

    = 10μs + 8ms + 50% × 15μs + 50% × 0

    = 10μs + 8000μs + 7.5μs = 8017.5μs

    EAT = (1-0.05) × 0.2μs + 0.05 × 8017.5μs

    = 0.19 + 400.875 = 401.065μs

    常考易错点总结
  • FIFO存在Belady异常，LRU和OPT不会（它们是栈算法）
  • 缺页中断在指令执行期间产生，一条指令可能引发多次缺页中断
  • 改进型CLOCK最多4轮扫描一定能找到淘汰页，第1轮和第3轮会将A置0
  • 计算EAT时注意时间单位统一，1ms = 1000μs = 10⁶ns
  • "可变分配全局置换"缺页时优先从空闲块分配，不会立即淘汰本进程页面
  • 工作集决定的是最少需要多少物理块才不会抖动
  • 虚拟内存的核心特征是多次性，虚拟性是结果
  • 页面置换是在缺页且无空闲物理块时才发生，有空间直接调入无需置换
