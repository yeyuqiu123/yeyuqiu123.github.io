🔧 DMA 方式工作过程流程图

1
📟 I/O 设备
设备向 CPU 发送 DMA 请求
2
🖥️ CPU
CPU 向 DMA 控制器发送启动命令，并设置 MAR（内存地址寄存器）和 DC（数据计数器）的初值，随后启动 DMA 控制器
3
🖥️ CPU
CPU 将 I/O 控制权交予 DMA 控制器，转而执行其他任务
🖥️ CPU 并行执行
CPU 继续执行其他计算任务
（无需干预数据传输）
🔀 DMA 控制器执行
接管总线控制权
以突发方式将整块数据在设备与内存之间直接传输
4
🔀 DMA 控制器
数据传输完成后，向 CPU 发送中断信号，通知操作完成
💡 关键特点：DMA 方式实现了设备与内存之间的直接数据传输，整个过程无需 CPU 干预，仅在开始和结束时需要 CPU 参与，大大提高了系统效率。
